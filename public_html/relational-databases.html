Relational Databases

  • Relational databases were invented in the 1970s
  • Relational databases are now the storehouse of mountains of data available to businesses

ACID Properties

  • These properties basically mean that when data is stored or updated in a database, it really is changed
  • Databases have transaction logs and other capabilities to ensure that changes happen and the modified data is visible when the data modification steps completes
  • In practice, databases support
    • Transactions
    • Logs
    • Replication
    • Concurrent Access
    • Stored Procedures
    • Security
    • Designing real-world applications
  • Atomicity
  • Consistency
  • Isolation
